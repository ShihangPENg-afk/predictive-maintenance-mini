## Overview

Binary classification with scikit-learn: predict batch quality (`normal` / `defect`) from line sensor readings (temperature, pressure, vibration, speed, humidity).

| Stage | Script / Component | Output |
|-------|-------------------|--------|
| EDA | `scripts/eda.py` | `docs/eda_summary.md` |
| Training | `scripts/train_model.py` | `artifacts/model.pkl`, `metrics.json`, `schema.json` |
| Experiment tracking | MLflow (SQLite) | `mlflow.db` (gitignored) |
| Inference API | `app/main.py` | `/health`, `/model-info`, `/predict` |
| Container | `Dockerfile` + `docker-compose.yml` | Port `8010` |

## Dataset

**Provenance**: The CSV in this repo is **reproducible simulated data** (not real factory data; no third-party copyright concerns). Generated by `scripts/generate_sample_data.py` using sensor distributions and a defect-risk rule (`random_state=42`, 500 rows).

Regenerate data:

```bash
python scripts/generate_sample_data.py
python scripts/eda.py
python scripts/train_model.py
```

File: `data/raw/manufacturing_quality.csv`

| Column | Type | Description |
|--------|------|-------------|
| `temperature` | float | Temperature |
| `pressure` | float | Pressure |
| `vibration` | float | Vibration |
| `speed` | float | Line speed |
| `humidity` | float | Humidity |
| `target` | str | Label: `normal` / `defect` |

Size: **500 rows × 6 columns**, no missing values. Class imbalance (~65% normal, ~35% defect). See `docs/eda_summary.md` and `docs/experiment_report.md`.

## EDA

```bash
git clone https://github.com/ShihangPENg-afk/predictive-maintenance-mini.git
cd predictive-maintenance-mini
pip install -r requirements.txt
python scripts/eda.py
```

`scripts/eda.py` reads the CSV, detects the target column, prints shape/types/missing values, summarizes the target distribution and numeric features, and writes `docs/eda_summary.md`.

## Training

```bash
python scripts/train_model.py
```

Pipeline (`scripts/train_model.py`):

1. Load CSV; drop rows with empty target
2. Split numeric vs categorical features (all numeric in current dataset)
3. Preprocessing: median impute + scale (numeric); most-frequent impute + one-hot (categorical, reserved)
4. **Current baseline**: `RandomForestClassifier` (`n_estimators=200`, `class_weight="balanced"`, `random_state=42`). **XGBoost / LightGBM are not implemented yet** (see Roadmap)
5. 80/20 stratified split
6. Metrics: `accuracy`, `f1_macro`, full classification report
7. Save to `artifacts/` and log one MLflow run

Or:

```bash
make train
```

## MLflow

Runs are stored in local SQLite `mlflow.db` (gitignored):

- Experiment: `predictive-maintenance-mini`
- Params: `model_type`, `n_estimators`, `test_size`, `random_state`, `target_column`
- Metrics: `accuracy`, `f1_macro`
- Artifacts: `model.pkl`, `metrics.json`, `schema.json`

```bash
bash scripts/start_mlflow_ui.sh
```

Default UI: http://127.0.0.1:5001

## FastAPI Service

Requires trained `artifacts/model.pkl`:

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8010 --reload
```

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/health` | GET | Health check |
| `/model-info` | GET | Features, classes, training metrics |
| `/predict` | POST | Single-sample inference |

Request body:

```json
{
  "features": {
    "temperature": 73.5,
    "pressure": 5.2,
    "vibration": 2.1,
    "speed": 118.0,
    "humidity": 48.0
  }
}
```

Response includes `prediction`, `prediction_label`, `probabilities`, `risk_level` (high / medium / low), and a text `recommendation`.

## Results (baseline)

RandomForest on 500 rows, 80/20 stratified split (`artifacts/metrics.json`):

| Metric | Value |
|--------|-------|
| accuracy | 0.71 |
| f1_macro | 0.67 |
| defect recall | 0.51 (imbalanced classes) |

Full report: [`docs/experiment_report.md`](docs/experiment_report.md). **Not for production decisions.**

## Limitations

- Demo / POC only; not validated for production
- Baseline only; no hyperparameter search; not targeting SOTA
- Small simulated dataset (500 rows)
- Low defect recall under class imbalance
- Static `artifacts/` only; no online retraining or A/B deployment

## Integration with rag-agentic-system
